FAQ

What is the intention of this site
The goal of the VRIF Test assets database is to establish a framework such that the VRIF interoperability ecosystem can grow in scale. This includes:
  • Test contents/Source content collection
  • Test vectors for the test contents.
  • Dynamic management of the attributes of all the models

What is the structure of the datatase
The database is structured as follows. Each testvector is assigned to one or more testcontents. All of the aforementioned entities have dynamic attributes. This means authorized users can dynamically add and remove attributes from testcontents and testvectors.

How to use the page

The main purpose of this page is to give a structured overview of the available testvectors. The list of the current testcontents and testvectors is accessible for everyone. In addition, authorized users are allowed to add, edit and remove entries for each of the entities.

The list view offers different contents related to attribute selection and search and filter functionality. It is possible to toggle certain columns in order to show less information in the respective tables. Moreover a column specific search can be performed for each of the attributes.

How can I add / edit / remove new elements
Add, edit and remove functions are only available for authorized users. In order to add a new element select the type of the element in left menu and click on "Create". Fill out the form and click on "Submit". To edit or remove an element select "My elements" and click on edit/remove. You might need to scroll to the right in order to see the specific buttons.

How can I add /edit /remove attributes
Add, edit and remove functions are only available for authorized users. In order to add a new attribute select "Attributes" and click on "Create". Fill out the form and click on "Submit". To edit or remove an element select "My Attributes" and click on edit/remove.

Each attribute has a specific type to which it belongs. Moreover, attributes can be set to inactive in which case they will not be visible in the list of the specific type of element they belong to. Furthermore, you can assign a default value for each attribute. Existing instances of the specific type of element will automatically be populated with the new attribute and the default value.

How can I report issues

If any issue is noticed, please file an issue report , including an informal description of the issue (GitHub user account required for creating issue). Github link is provided at the bottom of the main page.
